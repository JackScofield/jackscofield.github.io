i) Q1.33

Give an efficient algorithm to compute the least common multiple of two n-bit numbers x and y, that is, the smallest number divisible by both x and y. What is the running time of your algorithm as a function of n?

we know that every natural number can be written as form of products of prime numbers. So x=p1∗p2∗...Pnx=p_1*p_2*...P_nx=p1​∗p2​∗...Pn​, y=q1∗q2∗...qny=q_1*q_2*...q_ny=q1​∗q2​∗...qn​ .

lcm(x,y)=x∗ygcd(x,y)lcm(x,y)=\frac{x*y}{gcd(x,y)}lcm(x,y)=gcd(x,y)x∗y​

my algorithm is:

def gcd(x, y):  # Euclid's algorithm
    # a > b
    x, y = (x, y) if x >= y else (y, x)
    while y:
        x, y = y, x % y
    return x


def lcm(x, y):
    return (x * y) // gcd(x, y)

Running time:

Finding x*y takes O(n^2) time,

Finding gcd(x,y) takes O(n^3) time,

Division also takes O(n^2) time.

Therefore overall running time should be O(n^3)

ii) Q1.39

Give a polynomial-time algorithm for computing abca^{b^c}abc mod p, given a, b, c, and prime p.

Since p is a prime number, then p should pass Fermat test. We can use Fermat little theorem

$a^{p-1} \equiv 1 $ mod p

which means that

ap−1a^{p-1}ap−1 mod p = 1

Then we actually need to solve the question below:

abc=abc mod (p−1)a^{b^c}=a^{b^c \text{ mod } (p-1)}abc=abc mod (p−1) mod p

First we need to compute bc mod (p−1)b^c \text{ mod } (p-1)bc mod (p−1) and according to Fig 1.4 which is a Modular exponentiation algorithm. Thus it takes O(n^3) time.(n is the number of bit of a,b,c)

Second, if bc mod (p−1)b^c \text{ mod } (p-1)bc mod (p−1) is equal to t, then we need to compute ata^tat mod p, which is also a modular exponentiation algorithm, so O(n^3)

Therefore, overall running time should be O(n^3) which is polynomial.

iii) Q1.40

Show that if x is a nontrivial square root of 1 modulo N, that is, if x^2 ≡ 1 mod N but x ≡ ±1 mod N, then N must be composite. (For instance, 4^2 ≡ 1 mod 15 but 4 6≡ ±1 mod 15; thus 4 is a nontrivial square root of 1 modulo 15.)

Assume that N is prime and x is a nontrivial square root of 1 modulo N. x2x^2x2 mod N = 1, so x2x^2x2=kN+1 k is a natural number.

x2−1x^2-1x2−1=kN

(x-1)(x+1)=kN

Since N is prime, and every natural number is a product of prime numbers, so for (x-1) and (x+1) , at least one of them is divisible by N.

Thus, x≡±1x\equiv \pm 1x≡±1 mod N which contradicts to the property of nontrivial square root.

Hence, N must be composite.

iv) Q1.42

Suppose that instead of using a composite N = pq in the RSA cryptosystem (Figure 1.9), we simply use a prime modulus p. As in RSA, we would have an encryption exponent e, and the encryption of a message m mod p would be me mod p. Prove that this new cryptosystem is not secure, by giving an efficient algorithm to decrypt: that is, an algorithm that given p, e, and me mod p as input, computes m mod p. Justify the correctness and analyze the running time of your decryption algorithm.

If we use p instead of pq, N=p, then for e, e only need to satisfy 1<e<p-1 and e is relatively prime to p-1.

we choose d when ed mod p-1 =1 ( d is inverse of e mod (p-1)) we can generate d by using Euclid's algo.

decryption algo would be (xex^exemod p)^d mod p

This is not secure.

Suppose Alice wants to send message to Bob. Using public key (p,e).

Jack wants to intercept and decrypt the message, and Jack has the public key (p,e). Jack only need to calculate d by using the method ed mod (p-1) = 1. Then Jack has the private key(p,d) which should only belong to Bob.

Running time: it is mostly about generating d by using Euclid's Algorithm. So O(n^3)
